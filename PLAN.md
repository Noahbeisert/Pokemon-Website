# Pokemon Champions — Website Plan

> **Format: VGC Doubles only.** This tool is built exclusively for the doubles format (bring 4 of 6, pick 2 each turn). Singles mechanics, clauses, and meta are out of scope.

## Architecture

**Pure frontend: HTML + CSS + JS + JSON. No backend required.**

`pokemon_db.json` (generated by the PyCharm project's `db_example.py`) is the single data source.
Load it once via `fetch('pokemon_db.json')` and keep it in memory for all pages.

The PHP files can be deleted or ignored — everything runs client-side.

## Page 2: team-builder.html

### Purpose
Pick your 6 pokemon. See a threat matrix showing what the opponent's pokemon could threaten you with, and what you can hit for SE.

### Layout
```
[ Search/autocomplete input ]
[ Your Team: 6 slots (empty until filled) ]

--- after team is filled ---

[ Threat Matrix table ]
```

### Team Slots
- 6 slots, each shows: pokemon sprite (from PokeAPI), name, types as pills
- Click a slot to remove it
- Autocomplete searches by name from pokemon_db.json

### Sprite URL pattern (PokeAPI, free, no auth):
```
https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/{dex_id}.png
```
Use `dex_id` from pokemon_db.json.

### Threat Matrix
After 6 pokemon are selected, render a table:

**Rows:** your 6 pokemon (name + type pills)
**Columns:** each of your pokemon's weaknesses (types that hit them 2x or 4x)

For each cell: show which of YOUR OTHER pokemon could revenge kill (has a move of that type).
Color coding:
- Red cell = that pokemon is very threatened (4x weak)
- Orange cell = 2x weak
- Green = you have a counter in your team

Also below the table: "Coverage gaps" — types in the opponent meta that you have NO answer for.

### Data needed from pokemon_db.json per pokemon:
```json
{
  "name": "Incineroar",
  "types": ["Fire", "Dark"],
  "weak4x": [],
  "weak2x": ["Water", "Fighting", "Ground", "Rock"],
  "resist": [...],
  "immune": ["Psychic"],
  "moves": [{"name": "Fake Out", "type": "Normal"}, ...],
  "niche_threat": {"move": "Earthquake", "type": "Ground", "note": "..."},
  "tier_doubles": "S",
  "roles": ["fake_out", "intimidate", "pivot"]
}
```

## Page 3: preview.html (Battle Preview — most important page)

### Purpose
During team preview in a real match, quickly type in the opponent's 6 pokemon.
Get instant analysis:
1. What they will likely **lead** with (top 2 predictions with %)
2. What 4 they will likely **bring** to the field
3. For each of their pokemon: what it threatens on YOUR team
4. For each of YOUR pokemon: what theirs threatens against you (so you know when to switch/protect)

### Layout — Two phases

**Phase 1: Input**
```
Your Team (pre-loaded or quick-fill, 6 slots)
Opponent Team (6 search inputs, autocomplete)
[ Analyze button ]
```

**Phase 2: Analysis (shown after Analyze is clicked)**

#### Section A: Lead Prediction
Two large cards side by side:
```
[ Most likely lead #1 ]    [ Most likely lead #2 ]
  Incineroar (72%)           Whimsicott (68%)
  Fire/Dark                  Grass/Fairy
  Fake Out + Intimidate      Tailwind setter
  Reason: fake_out role,     Reason: tailwind role,
  S-tier                     paired with sweepers
```

#### Section B: Likely Bring (4 from 6)
Which 4 of their 6 are most likely to actually be brought.
Scoring: tier weight + coverage vs YOUR team.

Show as a ranked list: top 4 highlighted, bottom 2 dimmed.

#### Section C: Threat Matrix
Grid: **Their pokemon (rows) × Your pokemon (columns)**

Each cell: effectiveness of their moves against your pokemon.
- Dark red = their pokemon has a move that hits yours 4x
- Red = 2x super effective move
- Green = your pokemon resists or is immune
- Gray = neutral

On hover/tap: show which specific move causes the threat.
Example: hover cell [Garchomp vs Incineroar] → "Earthquake (Ground) hits 2x"

#### Section D: Per-Pokemon Quick Guide
For each of YOUR pokemon, a small card:
```
Your Incineroar
  THREATENED BY: Garchomp (EQ x2), Excadrill (EQ x2), Tyranitar (Stone Edge x2)
  SAFE AGAINST:  Kingambit (resists Dark), Dragapult (resists Fire/Dark)
  YOUR ANSWERS:  Use vs Whimsicott (Dark hits 2x), Sneasler (Fire hits 2x)
```

## Lead Probability Algorithm

```javascript
function scoreLeads(opponentTeam, yourTeam) {
  const TIER_WEIGHT = { S: 5, "A+": 4, A: 3, B: 2, C: 1 };
  const LEAD_ROLE_BONUS = {
    fake_out:       3,   // almost always leads
    weather_setter: 2.5, // nearly guaranteed if team has 2+ weather abusers
    trick_room:     2,   // likely lead if team has 2+ slow pokemon
    tailwind:       2,   // speed control
    intimidate:     1.5, // weakens physical attackers
    redirection:    1.5, // Follow Me / Rage Powder
    hazard_setter:  0.5, // sometimes leads
    sweeper:        0,   // rarely leads
    pivot:          0.5,
  };

  return opponentTeam.map(p => {
    let score = TIER_WEIGHT[p.tier_doubles] || 1;

    // Role bonuses
    (p.roles || []).forEach(role => {
      score += LEAD_ROLE_BONUS[role] || 0;
    });

    // Coverage bonus: how many of YOUR pokemon does this threaten on turn 1?
    // A pokemon that threatens 2+ of yours has more reason to lead
    const threatens = yourTeam.filter(yours =>
      p.moves.some(move =>
        yours.weak2x.includes(move.type) || yours.weak4x.includes(move.type)
      )
    ).length;
    score += threatens * 0.4;

    return { pokemon: p, score };
  })
  .sort((a, b) => b.score - a.score)
  .map((entry, i, arr) => {
    const total = arr.reduce((s, e) => s + e.score, 0);
    return { pokemon: entry.pokemon, probability: Math.round(entry.score / total * 100) };
  });
}
```

## Bring Prediction Algorithm

```javascript
function scoreBring(opponentTeam, yourTeam) {
  const TIER_WEIGHT = { S: 4, "A+": 3, A: 2.5, B: 2, C: 1 };

  return opponentTeam.map(p => {
    let score = TIER_WEIGHT[p.tier_doubles] || 1;

    // Coverage score: how many of YOUR pokemon does this threaten?
    const coverage = yourTeam.filter(yours =>
      p.moves.some(move =>
        yours.weak2x.includes(move.type) || yours.weak4x.includes(move.type)
      )
    ).length;
    score += coverage * 0.8;

    // Synergy: does this complement other high-score pokemon on their team?
    // (simplified: if they have weather setter, weather abusers score higher)
    const teamHasWeather = opponentTeam.some(x => x.roles?.includes('weather_setter'));
    if (teamHasWeather && p.roles?.includes('sweeper')) score += 0.5;

    const teamHasTR = opponentTeam.some(x => x.roles?.includes('trick_room'));
    if (teamHasTR && !p.roles?.includes('trick_room')) score += 0.3;

    return { pokemon: p, score };
  })
  .sort((a, b) => b.score - a.score);
  // Top 4 = predicted bring
}
```

## Shared JS: data.js

```javascript
// Load once, cache globally
let POKEMON = null;

async function loadData() {
  if (POKEMON) return POKEMON;
  const res = await fetch('pokemon_db.json');
  POKEMON = await res.json(); // keyed by lowercase name
  return POKEMON;
}

function getPokemon(name) {
  return POKEMON?.[name.toLowerCase()] || null;
}

function getAllPokemon() {
  return Object.values(POKEMON || {});
}
```

```javascript
const TYPE_COLORS = {
  Normal:"#A8A878", Fire:"#F08030", Water:"#6890F0", Electric:"#F8D030",
  Grass:"#78C850", Ice:"#98D8D8", Fighting:"#C03028", Poison:"#A040A0",

};

function typePill(type) {
  return `<span class="pill" style="background:${TYPE_COLORS[type]}">${type}</span>`;
}

function spriteUrl(dexId) {
  return `https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/${dexId}.png`;
}

// Autocomplete: filter pokemon by name prefix
function searchPokemon(query) {
  if (!query || query.length < 1) return [];
  const q = query.toLowerCase();
  return getAllPokemon()
    .filter(p => p.name.toLowerCase().includes(q))
    .sort((a, b) => {
      // Exact starts-with first
      const aStarts = a.name.toLowerCase().startsWith(q);
      const bStarts = b.name.toLowerCase().startsWith(q);
      if (aStarts && !bStarts) return -1;
      if (!aStarts && bStarts) return 1;
      return a.name.localeCompare(b.name);
    })
    .slice(0, 8);
}
```
